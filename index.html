Event Management System (My Latest Project)

Approved Events Table
(Client Side)

A table where approved events can be seen by the clients. This is also where clients can request for an event, which will be sent to the requests database, awaiting approval.

Client's Confirmation of Rejected Events

If an event is rejected, it will go straight to the rejected events table. Awaiting confirmation from clients, for the client to know that their event has been cancelled. After confirmation, it will immediately get deleted from the database.

Event Requests Table (Admin Side)

This table is only shown in the admin side of the webpage. This is where the client's request for an event goes to. Now the admin can either approve or reject the event, depending on their availability. If rejected it will show a form, asking why the event is rejected.

Approved Events Table
(Admin Side)

This table is the same table that is on the client's side, where the clients can see approved events. However, in the admin side, there are actions you can do to the events. You can edit the details of the event, you can delete the event if it is finished, and you can cancel the event. The difference between delete and cancelled, delete will delete the data entirely, while cancel will move it back to the requested events with the rejected status.

Rejected Events Table (Admin Side)

This table is where the rejected events are seen from the admin side. You can delete the table immediately to get rid of the data so that it doesn't take up the space on your table.

Users Table (Admin Side)

This table shows the users that are registered to your webpage, you can see their email, and password as seen. You can edit their details also to turn them into an admin or change details. You can also delete users.
